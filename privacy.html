Tressence OCR – Privacy Policy

Effective date: 2025‑11‑03

Overview

Tressence OCR is a Chrome extension that lets you extract text from images and PDF links on webpages, classify the text (e.g., name, place, phone, sentiment, topic), and optionally fill visible form fields on the page. This policy explains what data the extension processes, how it’s used, and your choices.

Data We Process

  • User‑selected content: Images or PDF pages you explicitly select (e.g., via right‑click menu). If a PDF has native text, the extension attempts to extract it locally via PDF.js; otherwise pages are rendered to images for OCR.
  • Derived outputs: OCR text and classification results generated for your use.
  • Local app data: Settings (templates, per‑site auto‑fill rules), recent results, quota counters, and overlay position are stored locally using Chrome storage.

How Your Data Is Used

  • To perform OCR and classification requested by you.
  • To display results in the extension UI and optionally fill visible form fields on the page when you choose.
  • To remember preferences and improve the user experience locally.

Third‑Party Services

The extension may send data you select (images/PDF pages or extracted text) to the APIs you configure, for the sole purpose of returning OCR/classification results to you:

  • Google Cloud Vision API (OCR): cloud.google.com/vision
  • DeepSeek API (classification): api.deepseek.com

The extension does not sell your data or share it for advertising.

Storage & Retention

  • No extension‑managed server storage. Results and settings are kept locally in Chrome storage on your device.
  • You can clear data at any time by removing the extension’s site data or uninstalling the extension.

Permissions Used

  • storage: Save settings, recent results, usage counters, overlay position.
  • contextMenus: Provide right‑click actions to start OCR/classification.
  • activeTab: Operate on the current tab after you trigger an action.
  • scripting: Inject minimal scripts to display the overlay and fill form fields that you choose.
  • offscreen: Render PDFs with PDF.js in an offscreen document due to MV3 limitations.
  • tabs: Determine a suitable web tab for filling and focus result windows.
  • host permissions: Allow the overlay and user‑initiated form filling to work on the sites where you act.

Children’s Privacy

This extension is not intended for children under 13.

Security

Communication with third‑party APIs uses HTTPS. We recommend configuring only trusted API providers.

Changes

We may update this policy. We will post any changes to this page and update the effective date.
